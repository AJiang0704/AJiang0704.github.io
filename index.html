系统综合实践-容器相关概念

一、微服务（Microservices）

微服务是一种软件架构风格，它是以专注于单一责任与功能的小型功能区块为基础，利用模块化的方式组合出复杂的大型应用程序，各功能区块使用与语言无关的API集相互通信。

优：

  1. 服务器的独立部署，耦合性底
  2. 服务的快速启动，因为依赖的库少了，代码量也少了
  3. 更加适合敏捷开发
  4. 职责专一，由专门的团队负责专门的服务
  5. 服务可以动态按需扩容
  6. 代码的复用，每个服务都提供REST、API

劣：

  1. 分布式部署，调用的复杂性高
  2. 独立的数据库，分布式事务的挑战
  3. 测试的难度提升，API文档的管理尤为重要
  4. 运维难度的提升

部署：

  1. 基于主机（物理机或虚拟机）的多服务实例
  2. 基于主机（物理机或虚拟机）的服务实例 基于单个虚拟机、单个容器
  3. 基于容器的服务实例
    将服务打包成一个文件系统类型的镜像（容器镜像），其中包含执行该服务所需的应用程序及其库文件，完成打包后，启动一到多个容器，并在物理或虚拟机上运行它们。
  4. 无服务器部署

二、Docker

镜像是构建Docker的基石，容器是基于镜像启动起来的。镜像是Docker生命周期中的构建或者打包阶段，而容器则是启动或执行阶段。

三、在Ubuntu上搭建Docker环境并实现基本操作

curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun #官方安装脚本

基本指令：

  • 列出镜像： docker iamges

  • 查看镜像详细信息：docker inspect

  • 删除镜像：docker rmi

  • 创建容器：docker run -itd --name=container_name image_name

    其中 container_name 是自定义的容器名称，image_name是镜像名称。

  • 查询容器：

    • 查看正在运行的容器： docker ps

    • 查看所有容器： docker ps -a

  • 修改容器： docker commit

  • 删除容器：docker rm

  • 拉取镜像：docker pull xxx/xxx

  • 上传镜像：docker push xxx/xxx

Go语言学习笔记之Slice

切片(Slice)是围绕动态数组的概念设计的，它的长度并不固定，可以追加元素并会在切片容量不足时进行扩容。使用这种结构，可以更方便地管理和使用数据集合。

一、初识Slice

arr := [...]int{0,1,2,3,4,5,6,7}
s := arr[2:6]              //s的值为[2,3,4,5]
s := arr[:6]               //s的值为[0,1,2,3,4,5]
s := arr[2:]               //s的值为[2,3,4,5,6,7]
s := arr[:]                //s的值为[0,1,2,3,4,5,6,7]
func updateSlice(s []int){
    s[0] = 100             //arr原始数据也会改变
}

Slice本身没有数据，是对底层array的一个view

二、Slice的扩展

arr := [...]int{0,1,2,3,4,5,6,7}
s1 := arr[2:6]             //s1的值为[2,3,4,5]
s2 := s1[3:5]              //s2的值为[5,6]

Slice可以向后扩展，但不可以向前扩展

三、向Slice添加元素

s2 := append(s1,10)

添加元素时如果超越cap，系统会重新分配更大的底层数组（如果原来的数组不用了，会被垃圾回收掉）

四、创建Slice

//Creating slice
var s []int
for i :=0 ; i < 100 ; i++{
    s = append(s, 2 * i + 1)
}
s := []int{2,4,6,8}
s := make([]int,16)
s := make([]int,10,32)     //len = 10，cap = 32

五、拷贝Slice

//Copying slice
copy(s2,s1)
printSlice(s2)

六、从Slice删除元素

//Deleting elements from slice
s = append(s[:3],s[4:]...) //删掉第4个元素，len减1，cap不变
//Popping from front
//front := s[0]
s = s[1:]
//Popping from back
//tail := s[len(s)-1]
s = append(:len(s)-1)
